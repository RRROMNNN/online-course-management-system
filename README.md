# Course Management System (Lab Project)

Навчальний проєкт з дисципліни **«Git-технологія командної розробки проєктів»**.  
Тема: **Система управління навчальними курсами**.

## Мета проєкту
Розробити базову структуру програмної системи для керування курсами, студентами та викладачами, використовуючи Git (гілки, коміти, злиття) та GitHub.

---

## Опис Solution
Solution містить опис основних інтерфейсів і класів системи управління навчальними курсами.  
Система призначена для:
- створення та перегляду курсів;
- реєстрації студентів на курси;
- зберігання інформації про викладачів;
- зміни статусів курсів (Active / Completed / Archived).

## Інтерфейси

### 1) ICourse (Курс)
Призначення: описує базову функціональність навчального курсу.

Методи:
- **getCourseInfo()** — повертає інформацію про курс (назва, опис, тривалість);
- **getStatus()** — повертає поточний статус курсу;
- **updateStatus(newStatus)** — змінює статус курсу.

---

### 2) IStudent (Студент)
Призначення: описує функціональність роботи з даними студента та реєстрацією на курси.

Методи:
- **getStudentInfo()** — повертає дані студента (ПІБ, email);
- **enrollCourse(course)** — реєструє студента на курс.

---

### 3) IInstructor (Викладач)
Призначення: описує функціональність викладача.

Методи:
- **getInstructorInfo()** — повертає інформацію про викладача (ПІБ, спеціалізація).

---

## Класи, що реалізують інтерфейси

### Course
Реалізує **ICourse**.  
Зберігає дані: назва, опис, тривалість, статус.  
Дозволяє переглядати інформацію про курс та змінювати його статус.

### Student
Реалізує **IStudent**.  
Зберігає дані: ПІБ, email, список курсів студента.  
Дозволяє реєструвати студента на курс і переглядати його дані.

### Instructor
Реалізує **IInstructor**.  
Зберігає дані: ПІБ, спеціалізація.  
Дозволяє переглядати інформацію про викладача.
